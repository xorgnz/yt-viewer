# Database Change Plan: Stable Database IDs

## Status

Draft for task 2.1 through task 2.3. This file proposes the schema, keys, indexes, foreign keys, stable ID sources, and production migration sequence. It is not approval for schema edits, migration scripts, or production database changes.

## Design Direction

- Use entity-based ID names consistently across owning and referencing tables.
- Keep YouTube-backed IDs where they already exist.
- Keep the profile key value, but rename the field to `profile_id`.
- Give virtual channels an opaque generated `vchannel_id`.
- Remove database-generated serial IDs from entity identities and relationship contracts.
- Replace them with stable domain-derived IDs or app-generated opaque IDs.
- Keep serial row IDs only for event or bookkeeping records whose identity is arbitrary and whose rows are naturally generated in sequence.

## Stable ID Sources

| Entity/table | Stable ID | Source rule | Backfill rule |
| --- | --- | --- | --- |
| `source_channels` | `src_channel_id` | Existing YouTube channel ID value. | Copy from `youtube_id`. |
| `videos` | `video_id` | Existing YouTube video ID value. | Copy from `youtube_id`. |
| `profiles` | `profile_id` | Existing profile key value. | Copy from `key`. |
| `virtual_channels` | `vchannel_id` | App-generated opaque ID. | Generate in deterministic row order by current `id`; retry on collision. |
| `virtual_channel_assignments` | `(src_channel_id, vchannel_id)` | Stable FK pair. | Join old generated IDs to parent stable IDs. |
| `virtual_channel_assignment_video_selections` | `(src_channel_id, vchannel_id, video_id)` | Stable FK tuple. | Join old generated IDs to parent stable IDs. |
| `video_flags` | `(video_id, profile_id)` | Stable FK pair. | Join old generated IDs to parent stable IDs. |
| `watch_history` relationships | `video_id`, `profile_id` | Stable FKs; keep private row `id`. | Join old generated IDs to parent stable IDs. |

## Virtual Channel ID Rule

`virtual_channels.vchannel_id` is immutable after creation and generated by application code.

Generation rule:

1. Generate 16 random lowercase base32 characters.
2. Prefix with `vc_`.
3. Attempt insert.
4. Enforce uniqueness in the database with `PRIMARY KEY (vchannel_id)`.
5. On duplicate-key error, generate a new ID and retry.

Recommended alphabet:

- `abcdefghjkmnpqrstvwxyz23456789`

## Composite Key Rules

| Table | Identity |
| --- | --- |
| `virtual_channel_assignments` | `(src_channel_id, vchannel_id)` |
| `virtual_channel_assignment_video_selections` | `(src_channel_id, vchannel_id, video_id)` |
| `video_flags` | `(video_id, profile_id)` |

Do not add standalone stable IDs for these rows unless implementation proves a composite key is impractical.

## Proposed Latest Schema Shape

### `source_channels`

Columns:

- `src_channel_id VARCHAR(255) NOT NULL`
- `title TEXT NOT NULL`
- `description TEXT DEFAULT NULL`
- `thumbnail_url TEXT DEFAULT NULL`
- `published_at BIGINT DEFAULT NULL`
- `last_refreshed_at BIGINT DEFAULT NULL`

Keys:

- `PRIMARY KEY (src_channel_id)`

### `videos`

Columns:

- `video_id VARCHAR(255) NOT NULL`
- `src_channel_id VARCHAR(255) NOT NULL`
- `title TEXT NOT NULL`
- `description TEXT DEFAULT NULL`
- `published_at BIGINT DEFAULT NULL`
- `duration_seconds INT DEFAULT NULL`
- `thumbnail_url TEXT DEFAULT NULL`
- `length_classification VARCHAR(32) DEFAULT 'unknown'`

Keys and constraints:

- `PRIMARY KEY (video_id)`
- `FOREIGN KEY (src_channel_id) REFERENCES source_channels(src_channel_id) ON DELETE CASCADE ON UPDATE CASCADE`
- `CHECK (length_classification IN ('long', 'short', 'unknown'))`

Indexes:

- `idx_videos_src_channel` on `(src_channel_id)`

### `virtual_channels`

Columns:

- `vchannel_id VARCHAR(255) NOT NULL`
- `name VARCHAR(255) NOT NULL`

Keys:

- `PRIMARY KEY (vchannel_id)`
- `UNIQUE (name)`

### `virtual_channel_assignments`

Columns:

- `src_channel_id VARCHAR(255) NOT NULL`
- `vchannel_id VARCHAR(255) NOT NULL`
- `mode VARCHAR(32) NOT NULL DEFAULT 'all'`
- `created_at BIGINT NOT NULL DEFAULT (UNIX_TIMESTAMP(CURRENT_TIMESTAMP(3)) * 1000)`
- `updated_at BIGINT NOT NULL DEFAULT (UNIX_TIMESTAMP(CURRENT_TIMESTAMP(3)) * 1000)`

Keys and constraints:

- `PRIMARY KEY (src_channel_id, vchannel_id)`
- `FOREIGN KEY (src_channel_id) REFERENCES source_channels(src_channel_id) ON DELETE CASCADE ON UPDATE CASCADE`
- `FOREIGN KEY (vchannel_id) REFERENCES virtual_channels(vchannel_id) ON DELETE CASCADE ON UPDATE CASCADE`
- `CHECK (mode IN ('all', 'long_only', 'selected_only'))`

Indexes:

- `idx_vca_vchannel` on `(vchannel_id)`
- optional `idx_vca_src_channel` on `(src_channel_id)` only if query plans justify it

### `virtual_channel_assignment_video_selections`

Columns:

- `src_channel_id VARCHAR(255) NOT NULL`
- `vchannel_id VARCHAR(255) NOT NULL`
- `video_id VARCHAR(255) NOT NULL`
- `review_state VARCHAR(32) NOT NULL DEFAULT 'not_yet_reviewed'`
- `created_at BIGINT NOT NULL DEFAULT (UNIX_TIMESTAMP(CURRENT_TIMESTAMP(3)) * 1000)`
- `updated_at BIGINT NOT NULL DEFAULT (UNIX_TIMESTAMP(CURRENT_TIMESTAMP(3)) * 1000)`

Keys and constraints:

- `PRIMARY KEY (src_channel_id, vchannel_id, video_id)`
- `FOREIGN KEY (src_channel_id, vchannel_id) REFERENCES virtual_channel_assignments(src_channel_id, vchannel_id) ON DELETE CASCADE ON UPDATE CASCADE`
- `FOREIGN KEY (video_id) REFERENCES videos(video_id) ON DELETE CASCADE ON UPDATE CASCADE`
- `CHECK (review_state IN ('included', 'ignored', 'not_yet_reviewed'))`

Indexes:

- optional `idx_vcavs_assignment` only if query plans justify it
- `idx_vcavs_video` on `(video_id)`
- `idx_vcavs_state` on `(review_state)`

### `profiles`

Columns:

- `profile_id VARCHAR(255) NOT NULL`
- `name TEXT NOT NULL`

Keys:

- `PRIMARY KEY (profile_id)`

### `video_flags`

Columns:

- `video_id VARCHAR(255) NOT NULL`
- `profile_id VARCHAR(255) NOT NULL`
- `ignored TINYINT NOT NULL DEFAULT 0`
- `watched TINYINT NOT NULL DEFAULT 0`
- `favorite TINYINT NOT NULL DEFAULT 0`
- `updated_at BIGINT NOT NULL DEFAULT (UNIX_TIMESTAMP(CURRENT_TIMESTAMP(3)) * 1000)`

Keys:

- `PRIMARY KEY (video_id, profile_id)`
- `FOREIGN KEY (video_id) REFERENCES videos(video_id) ON DELETE CASCADE ON UPDATE CASCADE`
- `FOREIGN KEY (profile_id) REFERENCES profiles(profile_id) ON DELETE CASCADE ON UPDATE CASCADE`

### `watch_history`

Columns:

- `id INT AUTO_INCREMENT PRIMARY KEY`
- `video_id VARCHAR(255) NOT NULL`
- `profile_id VARCHAR(255) NOT NULL`
- `session_started_at BIGINT NOT NULL`
- `last_updated_at BIGINT NOT NULL`
- `time_watched_seconds INT NOT NULL DEFAULT 0`

Keys:

- `PRIMARY KEY (id)`
- `FOREIGN KEY (video_id) REFERENCES videos(video_id) ON DELETE CASCADE ON UPDATE CASCADE`
- `FOREIGN KEY (profile_id) REFERENCES profiles(profile_id) ON DELETE CASCADE ON UPDATE CASCADE`

Indexes:

- `idx_history_profile` on `(profile_id, session_started_at DESC)`
- `idx_history_video` on `(video_id, session_started_at DESC)`
- `idx_history_video_profile_update` on `(video_id, profile_id, last_updated_at DESC)`

### `migration_history`

No change. Keep generated `id` as private migration bookkeeping.

### `_meta`

No change.

## Production Migration Sequence

1. Take a production backup.
2. Record row counts for all affected tables.
3. Preflight-check uniqueness/nullability for:
   - `source_channels.youtube_id`
   - `videos.youtube_id`
   - `profiles.key`
4. Add new stable-ID columns side by side as nullable columns.
5. Backfill direct IDs:
   - `source_channels.src_channel_id` from `source_channels.youtube_id`
   - `videos.video_id` from `videos.youtube_id`
   - `profiles.profile_id` from `profiles.key`
   - `virtual_channels.vchannel_id` from generated opaque IDs
   - `videos.src_channel_id` from `source_channels.src_channel_id`
   - `virtual_channel_assignments.src_channel_id` from `source_channels.src_channel_id`
   - `virtual_channel_assignments.vchannel_id` from `virtual_channels.vchannel_id`
   - `video_flags.video_id` from `videos.video_id`
   - `video_flags.profile_id` from `profiles.profile_id`
   - `watch_history.video_id` from `videos.video_id`
   - `watch_history.profile_id` from `profiles.profile_id`
6. Backfill `virtual_channel_assignment_video_selections` tuple columns by joining through old assignment and video IDs.
7. Validate non-null stable columns and matching row counts.
8. Add stable PKs, FKs, and indexes.
9. Update application code and tests to use stable IDs.
10. After validation passes, remove old generated relationship columns and obsolete indexes/FKs.
11. Keep only private generated `watch_history.id` and `migration_history.id`.
12. Run post-migration smoke checks.

## Verification Expectations

- confirm row counts match
- confirm every old relationship maps to a stable relationship
- confirm no stable relationship column is null
- confirm FK checks pass
- confirm:
  - viewer loads videos
  - viewer filters work
  - history records and updates
  - flags update
  - admin source refresh still links videos
  - admin virtual-channel assignments work
  - selected-only review state persists

## Approval Gate

No schema edits, migration scripts, or production database changes should begin until the user explicitly approves this plan.
